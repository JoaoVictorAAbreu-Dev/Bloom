# Bloom Privacy Policy

**Effective date:** July 3, 2026

Bloom is a local-first Android app for habits, routines, focus sessions, statistics, garden progress, and optional AI coaching.

## What Bloom stores

Bloom stores user data locally on the device, including:

- profile name and onboarding choices
- habits, completions, and streaks
- routine blocks
- Pomodoro sessions
- theme and Pomodoro preferences
- export snapshots generated by the user

## How Bloom protects data

Bloom uses local encryption for sensitive preference fields and habit free-text fields before persistence. Android backups are disabled for the app.

## Optional AI processing

Bloom Coach can work in two modes:

- **Local fallback:** guidance is generated on-device without sending user content to an external API.
- **Remote AI:** if the user enables AI and configures the feature, Bloom may send a minimized and sanitized prompt to the configured AI backend.

Bloom does not require an account or cloud sync to function.

## Exported data

If the user shares an export snapshot, the shared content contains only the data present in the local export JSON. The user controls when the export is generated, cleared, or shared.

## Third-party services

When remote AI is enabled, Bloom may contact the configured AI backend or Groq proxy depending on the build and environment settings.

## Data deletion

Users can reset local data from Settings. This clears app-managed local data on the device.

## Contact

For questions about Bloom privacy or data handling, review the technical documentation or the repository maintainers.
